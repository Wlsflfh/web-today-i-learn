### HTML

1. 나만의 갤러리에서 사진 9장은 HTML 태그 중 어떤 요소로 표현하는 것이 좋을까? 이유는 뭐라고 생각했는가?

`<img>` 태그를 사용하는 것이 적절하다고 생각합니다. `<img>` 태그는 HTML에서 이미지를 표현하기 위한 가장 기본적인 시맨틱 요소로, 이미지 자체를 의미적으로 나타낼 수 있기 때문입니다.

1. 뼈대 코드를 기반으로 이미지를 로드하는 방식은 크게 2가지로 나뉘는데 무엇이고 어떻게 사용하고 있는가?

제가 사용중인 방식은 로컬 경로입니다. 프로젝트 폴더 내에 저장된 파일을 불러오는 방식입니다. (e.g. `<img src="./images/profile.jpeg">`) 두번째 방식은 인터넷상에 호스팅된 이미지 주소를 직접 넣는 외부 URL 방식입니다. (e.g. `<img src="https://example.com/image.jpg">`)

1. 시맨틱 태그(예: `<header>`, `<main>`, `<section>`, `<footer>` 등)로 문서를 구분할 때, 어떤 기준으로 섹션을 나누고 태그를 적용하는 것이 좋을까?

페이지의 상단은 `<header>`, 내비게이션은 `<nav>`, 주요 콘텐츠는 `<main>`, 주제별 묶음은 `<section>`, 하단 정보는 `<footer>`. 이런 식으로 문서의 구조적 역할에 따라 나누는게 좋다고 생각합니다.

1. `<img>` 태그의 `alt` 속성은 왜 필요하며, 어떻게 작성하는 것이 좋을까?

`alt` 속성은 이미지가 로드되지 않았을 때 대체 텍스트를 제공하고, 스크린 리더를 사용하는 사용자에게 이미지 내용을 전달하기 위해 필요합니다.

1. 이미지를 로드할 때, 상대 경로(relative path)와 절대 경로(absolute path)를 혼용하면 어떤 문제가 발생할 수 있을까?

상대 경로와 절대 경로를 혼용하면 프로젝트를 다른 환경에 배포할 때 일부 이미지가 정상적으로 로드되지 않을 수 있습니다. 또한 경로 관리가 복잡해지고, 유지보수 시 혼란을 초래할 수 있습니다. 따라서 하나의 방식으로 통일하는 것이 좋다고 생각합니다.

1. `<meta>` 태그의 viewport 설정과 Open Graph 태그는 각각 어떤 역할을 하는가?

메타 태그는 모바일 환경에서 화면 크기를 조정하여 반응형 웹이 제대로 동작하도록 도와줍니다. Open Graph 태그는 웹 페이지를 SNS에 공유할 때 제목, 설명, 이미지 등을 미리보기 형태로 표시해주는 역할을 합니다.

### CSS

1. 나만의 갤러리에서 9장의 이미지를 보기 좋게 배치하려면, Flexbox와 CSS Grid 중 어떤 것이 더 적합할까? 그 이유는?

갤러리처럼 여러 이미지를 행과 열로 정렬해야 하는 경우에는 CSS Grid가 더 적합하다고 생각합니다. Grid는 2차원 레이아웃을 지원하기 때문에 3x3 형태의 구조를 직관적으로 구성할 수 있습니다. 반면 Flexbox는 1차원 레이아웃에 적합하여 이런 경우에는 Grid가 더 효율적이라고 생각합니다.

1. 태그 선택자, 클래스 선택자, ID 선택자는 각각 언제 사용하는 것이 적절할까?

태그 선택자는 전체적인 기본 스타일을 지정할 때 사용해야 합니다. 클래스 선택자는 여러 요소에 반복적으로 적용할 스타일을 정의할 때 사용합니다. ID 선택자는 특정한 하나의 요소에만 적용할 때 사용하는 것이 적절하다고 생각합니다.

1. CSS를 HTML 내에 직접 작성하는 것과 별도 파일로 분리하는 것의 차이는 무엇일까?

CSS를 HTML 내부에 직접 작성하면 코드가 길어지고 유지보수가 어려워지는 것 같습니다. 반면 별도의 CSS 파일로 분리하면 스타일을 재사용할 수 있고, 구조와 디자인을 분리하여 관리할 수 있기 때문에 더 효율적이라고 생각합니다.

### JavaScript

1. `document.getElementById()`와 `document.querySelector()`의 차이는 무엇인가?

`getElementById()` 는 특정 id 값을 가진 요소 하나만 선택할 수 있습니다. 반면 `querySelector()`는 CSS 선택자 방식으로 요소를 선택할 수 있어 클래스, 태그, id 등 다양한 방식으로 유연하게 사용할 수 있습니다.

1. 이벤트 리스너를 사용해서 "어떤 일이 일어났을 때 어떤 동작을 실행"하게 만들 수 있다. 자기소개 페이지에서 활용할 수 있는 이벤트에는 어떤 것들이 있을까?

이벤트 리스너를 활용하면 다양한 상호작용을 구현할 수 있습니다. 예를 들어 버튼 클릭 시 다크 모드를 전환하거나, 폼 제출 시 TIL 항목을 추가하는 기능을 구현할 수 있습니다. 또한 이미지 클릭 시 확대 효과를 주거나, 스크롤 시 애니메이션을 적용하는 것도 가능합니다.

1. `innerHTML`로 DOM을 조작할 때 주의해야 할 점은 무엇일까?

`innerHTML`은 문자열 형태로 HTML을 삽입하기 때문에 보안상 취약할 수 있으며, 특히 외부 입력값을 그대로 사용할 경우 XSS 공격 위험이 있습니다. 또한 기존 요소를 덮어쓸 수 있기 때문에 의도하지 않은 DOM 구조 변경이 발생할 수 있습니다.

### 배포

1. GitHub Pages로 배포한 페이지에서 TIL을 작성하면, 새로고침하거나 재배포했을 때 데이터가 유지될까? 그 이유는?

GitHub Pages로 배포한 페이지에서 TIL을 작성하더라도, 새로고침하거나 재배포를 하면 데이터는 유지되지 않습니다. 그 이유는 현재 구현된 방식이 JavaScript를 이용해 **브라우저의 메모리 상에서만 데이터를 추가하는 구조**이기 때문입니다. 페이지가 새로 로드될 때 초기 HTML 상태로 다시 렌더링되기 때문에 기존에 추가했던 TIL 내용은 모두 사라지게 됩니다.

1. JavaScript로 추가한 데이터를 영구적으로 저장하려면 어떤 방법이 있을까?

JavaScript로 추가한 데이터를 영구적으로 저장하기 위해서는 별도의 저장소가 필요합니다. 백엔드 서버를 구축하고 데이터베이스에 데이터를 저장하면, 사용자가 어디서 접속하더라도 데이터를 유지할 수 있습니다.
